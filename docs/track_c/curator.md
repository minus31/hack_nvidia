# NeMo Curator

> 대규모 데이터셋을 텍스트, 이미지, 비디오, 오디오 모달리티 전반에서 정제(Curate)하여 AI 모델 학습 품질을 높이는 오픈소스 플랫폼

- **공식 문서**: https://docs.nvidia.com/nemo/curator/latest/
- **최신 버전**: v1.1.0 (26.02)

---

## 1. NeMo Curator란?

NeMo Curator는 AI 모델 학습에 사용할 **대규모 데이터셋을 정제(curate)하는 플랫폼**이다.

원시 데이터(웹 크롤링, 아카이브 등)에서 시작하여 정제된 고품질 학습 데이터셋을 만드는 전체 파이프라인을 제공한다.

```
원시 데이터 → 로딩 → 정제 → 중복 제거 → 품질 평가 → 필터링 → 학습 데이터셋
```

### Data Designer와의 차이

| | NeMo Curator | NeMo Data Designer |
|---|---|---|
| **목적** | 기존 데이터를 정제/필터링 | 새로운 합성 데이터를 생성 |
| **입력** | 웹 크롤링, 아카이브 등 원시 데이터 | 스키마 정의(YAML/Python) |
| **핵심 기능** | 중복 제거, 품질 필터링, 언어 감지 | LLM 기반 텍스트/구조 생성 |
| **GPU 활용** | 대규모 분산 처리 가속 | LLM 추론 |

## 2. 지원 모달리티

NeMo Curator는 4가지 모달리티를 지원한다.

### 2.1 텍스트 (Text)

가장 성숙한 모달리티로, 전체 정제 파이프라인을 제공한다.

#### 데이터 소스

| 소스 | 설명 |
|---|---|
| Common Crawl | 웹 아카이브 데이터 |
| arXiv | 학술 논문 |
| Wikipedia | 백과사전 덤프 |
| Custom JSONL/Parquet | 사용자 커스텀 데이터 |

#### 처리 파이프라인

```
1. 로딩 (JSONL/Parquet/웹 아카이브)
   ↓
2. 텍스트 정제 (Unicode 정규화, 특수문자 제거)
   ↓
3. 언어 감지 (FastText, 176개 언어 지원)
   ↓
4. 중복 제거 (3가지 방식)
   ↓
5. 품질 평가 및 필터링
   ↓
6. 최종 데이터셋 출력
```

#### 중복 제거 (Deduplication) — 3가지 방식

| 방식 | 알고리즘 | 특징 |
|---|---|---|
| **Exact** | MD5 해시 | 완전히 동일한 문서 제거. 가장 빠름 |
| **Fuzzy** | MinHash + LSH | 유사한 문서 제거. 임계값 조절 가능 |
| **Semantic** | 임베딩 기반 | 의미적으로 유사한 문서 제거. 가장 정교 |

#### 품질 평가 도구

- **FastText 분류기**: 도메인, 안전성, 품질 등급 분류
- **휴리스틱 필터링**: 설정 가능한 임계값 기반 규칙
- **GPU 가속 분류기**: 대규모 분산 처리 지원
- **점수 기반 필터링**: 다양한 메트릭에 따른 필터링

### 2.2 이미지 (Image)

#### 처리 파이프라인

```
1. TAR 아카이브 로딩 (DALI GPU 가속)
   ↓
2. CLIP 임베딩 생성 (OpenAI ViT-L/14)
   ↓
3. 미적 품질 필터링 (Human preference 모델)
   ↓
4. NSFW 콘텐츠 감지 및 제거
   ↓
5. 의미적 중복 제거 (CLIP 임베딩 기반)
   ↓
6. 메타데이터 보존 및 리샤딩
```

#### 사용 모델

| 모델 | 용도 |
|---|---|
| OpenAI ViT-L/14 | 임베딩 생성 |
| CLIP 기반 분류기 | 다운스트림 분류 |
| Aesthetic Predictor | 미적 품질 평가 |

### 2.3 비디오 (Video)

#### 처리 파이프라인

```
1. 클리핑 (Fixed stride 또는 TransNetV2 장면 전환 감지)
   ↓
2. 트랜스코딩 (CPU/GPU, 품질 튜닝)
   ↓
3. 모션/미적 필터링
   ↓
4. 임베딩 생성 (Cosmos-Embed1)
   ↓
5. 의미적 클러스터링 기반 중복 제거
   ↓
6. 프레임 추출 및 캡션 생성 (Qwen 모델)
```

**아키텍처**: Ray 기반 분산 시스템, 오토스케일링, 오브젝트 스토어 최적화, 스트리밍 처리

### 2.4 오디오 (Audio)

#### 처리 파이프라인

```
1. 수집 및 검증 (FLEURS, 커스텀 매니페스트, 로컬 파일)
   ↓
2. 자동 음성 인식 (NeMo Framework ASR 모델)
   ↓
3. 품질 메트릭 산출 (WER, CER, 발화 속도)
   ↓
4. 필터링 (길이, 형식, 품질 기준)
   ↓
5. 텍스트 통합 (멀티모달 워크플로)
```

#### 품질 메트릭

| 메트릭 | 설명 |
|---|---|
| WER (Word Error Rate) | 단어 오류율 |
| CER (Character Error Rate) | 문자 오류율 |
| Duration | 발화 길이 |
| Speech Rate | 발화 속도 |

## 3. 합성 데이터 생성

Curator에도 합성 데이터 생성 기능이 포함되어 있다.

| 기능 | 설명 |
|---|---|
| 다국어 Q&A 쌍 생성 | LLM을 활용한 질의-응답 데이터 생성 |
| 텍스트 변환 | 기존 텍스트의 스타일/형식 변환 |
| 지식 추출 | 문서에서 구조화된 지식 추출 |
| Nemotron-CC 파이프라인 | NVIDIA 전용 합성 데이터 파이프라인 |

LLM 클라이언트는 NVIDIA API 또는 커스텀 엔드포인트를 지원한다.

## 4. 인프라 및 배포

### 실행 백엔드

| 백엔드 | 설명 |
|---|---|
| **Ray** | 기본 분산 컴퓨팅 엔진 (v1.1.0부터 Dask에서 전환) |
| **Ray Actor Pool** | 실험적 Actor 기반 실행 |
| **컨테이너** | Docker 환경 지원 |

### 주요 아키텍처 구성 요소

| 구성 요소 | 설명 |
|---|---|
| **Pipeline** | 스테이지 순서를 관리하는 메인 오케스트레이터 |
| **Task Types** | DocumentBatch, ImageBatch, VideoTask, AudioBatch |
| **Executors** | XennaExecutor (프로덕션), Ray 백엔드 (실험) |
| **Resource Config** | CPU/GPU 요구사항 정의 |

### 특징

- **재개 가능**: 중단된 대규모 처리를 체크포인트에서 재개
- **모니터링**: Prometheus/Grafana 통합 지원
- **메모리 최적화**: GPU 메모리 관리 전략 내장
- **176개 언어 지원**: FastText 모델 기반 다국어 처리
