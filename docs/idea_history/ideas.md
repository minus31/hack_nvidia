## Idea 1. EvalOps — 도메인 전문가 협업 데이터 고도화 프레임워크

### 한줄 요약
NeMo Data Designer + Curator 위에 **도메인 전문가(현업)와 엔지니어가 함께 사용하는 협업 기반 데이터 제작·평가 도구**를 만든다.

### 문제 정의
- 현재 합성 데이터 제작은 **엔지니어 중심**: 프롬프트 설계, 품질 평가, 파이프라인 운영 모두 엔지니어가 담당
- 도메인 전문가(의사, 변호사, 교사 등)는 **데이터 품질을 판단할 수 있지만** 기술 도구에 접근하기 어려움
- 피드백 루프가 느림: 전문가 리뷰 → 엔지니어에게 전달 → 프롬프트 수정 → 재생성 → 다시 리뷰

### 핵심 아이디어
```
[도메인 전문가]                    [엔지니어]
    │                                │
    ├─ 샘플 데이터 리뷰 (UI)         ├─ 파이프라인 설계 (Code)
    ├─ 품질 기준 정의 (자연어)        ├─ 프롬프트 템플릿 작성
    ├─ 좋은/나쁜 예시 태깅           ├─ LLMJudge 기준 반영
    └─ 평가 결과 확인               └─ Curator 필터링 규칙 설정
         │                              │
         └──────── EvalOps ─────────────┘
                      │
              NeMo Data Designer + Curator
```

### 주요 기능 구상
1. **리뷰 대시보드**: 생성된 데이터 샘플을 전문가가 직관적으로 리뷰
2. **자연어 기반 평가 기준 정의**: "풀이 과정에 단위가 빠진 것은 감점" → LLMJudge 프롬프트로 자동 변환
3. **피드백 루프 자동화**: 전문가 태깅(좋음/나쁨) → 프롬프트 개선 제안 → 재생성
4. **품질 메트릭 추적**: 라운드별 품질 점수 변화 시각화

### 기술 스택
- NeMo Data Designer: 합성 데이터 생성 파이프라인
- NeMo Curator: 품질 필터링, 중복 제거
- Nemotron 모델: 데이터 생성 + 평가(Judge)
- 프론트엔드: Streamlit 또는 Gradio (데모용)

### 평가
| 항목 | 평가 |
|---|---|
| Track C 적합성 | ★★★★★ — 합성 데이터 파이프라인의 직접적 확장 |
| 차별점 | ★★★★☆ — "도메인 전문가 협업"은 현재 도구 생태계에서 부족한 영역 |
| 데모 가능성 | ★★★★☆ — Streamlit으로 인터랙티브 데모 제작 가능 |
| 해커톤 구현 난이도 | ★★★☆☆ — UI + 백엔드 + 파이프라인 연동 필요, 스코프 관리 중요 |
| 실용성 | ★★★★★ — 실제 기업 현장에서 바로 필요한 도구 |

## Idea 2. Self-Evolving Prompt Optimization for SDG

### 한줄 요약
**자기 진화(Self-Evolving) 알고리즘**을 적용해 합성 데이터 생성 프롬프트를 자동으로 최적화하는 방법론 + 이를 구현한 도구.

### 문제 정의
- 합성 데이터의 품질은 **프롬프트 품질에 크게 의존**
- 현재 프롬프트 엔지니어링은 수동 시행착오(trial-and-error) 기반
- 데이터 도메인이 바뀔 때마다 프롬프트를 처음부터 다시 설계해야 함
- LLMJudge 점수를 최적화 신호로 활용할 수 있지만, 이를 자동 루프로 연결하는 도구가 없음

### 핵심 아이디어
```
[초기 프롬프트]
      │
      ▼
[Data Designer로 데이터 생성 (N개)]
      │
      ▼
[LLMJudge로 품질 평가]
      │
      ▼
[평가 결과 분석 — 어떤 패턴이 고품질/저품질인지]
      │
      ▼
[Self-Evolving 알고리즘으로 프롬프트 변이(mutation) + 선택(selection)]
      │
      ▼
[개선된 프롬프트로 재생성]
      │
      ▼
[반복... 수렴할 때까지]
```

### 적용 가능한 Self-Evolving 접근법 (조사 필요)
| 접근법 | 설명 | 적합성 |
|---|---|---|
| **OPRO** (Optimization by PROmpting) | LLM 자체를 옵티마이저로 사용, 이전 프롬프트+점수를 보고 더 나은 프롬프트 제안 | ★★★★★ |
| **EvoPrompt** | 진화 알고리즘(GA/DE)으로 프롬프트 변이·교차·선택 | ★★★★☆ |
| **DSPy** | 프로그래밍적 프롬프트 최적화 프레임워크 | ★★★☆☆ |
| **Self-Refine** | LLM이 자기 출력을 비판하고 개선하는 반복 루프 | ★★★★☆ |
| **ProTeGi** | 텍스트 그래디언트 기반 프롬프트 최적화 | ★★★☆☆ |

### 기술 스택
- NeMo Data Designer: 데이터 생성 (inner loop)
- Nemotron 모델: 생성 + 평가 + 프롬프트 최적화
- Self-Evolving 엔진: OPRO 또는 EvoPrompt 구현
- NeMo Curator: 최종 데이터 정제

### 평가
| 항목 | 평가 |
|---|---|
| Track C 적합성 | ★★★★★ — SDG 파이프라인의 핵심 개선 |
| 차별점 | ★★★★★ — 프롬프트 자동 최적화는 연구적 가치 높음 |
| 데모 가능성 | ★★★★☆ — "라운드별 품질 향상 그래프"로 직관적 시각화 가능 |
| 해커톤 구현 난이도 | ★★★★☆ — 알고리즘 구현 + 실험 반복 시간 필요 |
| 실용성 | ★★★★☆ — 범용적으로 적용 가능하나 수렴성 보장이 관건 |

## Idea 3. Visual Prompt Optimization Dataset

### 한줄 요약
이미지/비디오 생성 AI를 위한 **프롬프트 생성기(Prompt Generator)** 학습용 **Image-Text 합성 데이터셋**을 제작하는 파이프라인.

### 문제 정의
- Stable Diffusion, DALL-E, Sora 등 이미지/비디오 생성 모델이 대중화
- 하지만 **원하는 결과를 얻기 위한 프롬프트 작성이 어려움** (프롬프트 엔지니어링 진입장벽)
- 기존 프롬프트 생성기는 범용적이어서 **특정 스타일/도메인에 최적화되지 않음**
- 프롬프트 생성기를 학습/최적화하려면 **고품질 Image-Text 쌍 데이터셋**이 필요

### 핵심 아이디어
```
[목표 도메인 정의] (예: 건축 시각화, 제품 사진, 게임 컨셉아트)
      │
      ▼
[기존 고품질 이미지 수집 + 캡셔닝]
      │
      ▼
[NeMo Data Designer로 합성 프롬프트 생성]
  ├─ 스타일 변형: "cyberpunk", "watercolor", ...
  ├─ 구도 변형: "close-up", "aerial view", ...
  └─ 디테일 변형: 조명, 배경, 분위기, ...
      │
      ▼
[이미지 생성 모델로 이미지 렌더링]
      │
      ▼
[품질 평가 — CLIP Score, 미적 점수, 프롬프트 충실도]
      │
      ▼
[NeMo Curator로 데이터 정제 (중복 제거, 품질 필터링)]
      │
      ▼
[최종 Image-Text 데이터셋]
  → 프롬프트 생성기 학습/파인튜닝에 사용
```

### 데이터셋 구조 (예시)
| 컬럼 | 설명 |
|---|---|
| `user_intent` | 사용자의 자연어 의도 ("귀여운 고양이 일러스트") |
| `optimized_prompt` | 생성 모델에 최적화된 프롬프트 |
| `image_url` | 생성된 이미지 |
| `clip_score` | 프롬프트-이미지 정합도 |
| `aesthetic_score` | 미적 품질 점수 |
| `style_tags` | 스타일 태그 |
| `domain` | 도메인 카테고리 |

### 기술 스택
- NeMo Data Designer: 프롬프트 합성 생성
- Nemotron 모델: 프롬프트 생성 + 품질 평가
- NeMo Curator: 이미지 모달리티 정제 (CLIP 임베딩, 미적 필터링)
- 이미지 생성 모델: SDXL / Flux (평가용)

### 평가
| 항목 | 평가 |
|---|---|
| Track C 적합성 | ★★★☆☆ — 합성 데이터셋 제작이지만, 이미지 생성 모델 의존도 높음 |
| 차별점 | ★★★★☆ — 멀티모달 데이터셋이라는 점에서 독특 |
| 데모 가능성 | ★★★★★ — 이미지가 포함되어 시각적으로 임팩트 있음 |
| 해커톤 구현 난이도 | ★★★★★ — 이미지 생성 인프라 + 평가 파이프라인 구축 필요, 범위 넓음 |
| 실용성 | ★★★★☆ — 크리에이터 시장 수요 높지만, 니치한 영역 |
