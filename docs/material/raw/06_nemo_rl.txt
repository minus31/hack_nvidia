3. 데이터 준비

GRPO는 프롬프트와 정답 쌍을 JSONL 형식으로 입력받으며, 모델이 생성한 응답을 reward function이 정답과 비교해 보상을 계산합니다. 커스텀 데이터셋을 사용할 경우 input_key / output_key로 필드명을 지정하면 됩니다 (기본값: "input", "output").

3.1 DatumSpec — 학습 샘플 구조

모든 데이터 프로세서는 DatumSpec 형식을 반환해야 합니다. (소스 코드)

python
Copy
# nemo_rl/data/interfaces.py
from typing import Any, Union
from typing_extensions import NotRequired, TypedDict
import torch

LLMMessageLogType = list[dict[str, Union[str, torch.Tensor]]]

class DatumSpec(TypedDict):
    message_log: LLMMessageLogType       # 대화 히스토리 (role, content, token_ids)
    length: int                          # 전체 토큰 시퀀스 길이
    extra_env_info: dict[str, Any]       # reward function에 전달될 정답 등 추가 정보
    loss_multiplier: float               # 손실 가중치 (시퀀스가 너무 길면 0.0으로 마스킹)
    idx: int                             # 샘플 인덱스
    task_name: NotRequired[str]          # 태스크 식별자 (기본값: "default")
    stop_strings: NotRequired[list[str]] # 생성 중단 문자열 (선택)
    __extra__: NotRequired[Any]          # 추가 필드 허용
3.2 커스텀 데이터 프로세서 작성

NeMo RL에서 제공하는 math_data_processor를 참고해 자체 프로세서를 작성합니다. (소스 코드)

python
Copy
# nemo_rl/data/processors.py — math_data_processor
from typing import Any, cast
import torch
from nemo_rl.data.interfaces import DatumSpec, LLMMessageLogType, TaskDataSpec

def math_data_processor(
    datum_dict: dict[str, Any],
    task_data_spec: TaskDataSpec,
    tokenizer,
    max_seq_length: int,
    idx: int,
) -> DatumSpec:
    problem  = datum_dict["problem"]
    solution = str(datum_dict["expected_answer"])
    extra_env_info = {"ground_truth": solution}

    message_log: LLMMessageLogType = []

    # 시스템 프롬프트 처리
    if task_data_spec.system_prompt:
        sys_prompt: dict[str, str | torch.Tensor] = {
            "role": "system",
            "content": task_data_spec.system_prompt,
        }
        sys = tokenizer.apply_chat_template(
            [cast(dict[str, str], sys_prompt)],
            tokenize=False,
            add_generation_prompt=False,
            add_special_tokens=False,
        )
        sys_prompt["token_ids"] = tokenizer(
            sys, return_tensors="pt", add_special_tokens=False
        )["input_ids"][0]
        message_log.append(sys_prompt)

    # 유저 메시지 처리
    if task_data_spec.prompt:
        problem = task_data_spec.prompt.format(problem)
    user_message = {"role": "user", "content": problem}
    message = tokenizer.apply_chat_template(
        [user_message],
        tokenize=False,
        add_generation_prompt=True,
        add_special_tokens=False,
    )
    user_message["token_ids"] = tokenizer(
        message, return_tensors="pt", add_special_tokens=False
    )["input_ids"][0]
    user_message["content"] = message
    message_log.append(user_message)

    length = sum(len(m["token_ids"]) for m in message_log)

    # max_seq_length 초과 시 마스킹
    loss_multiplier = 1.0
    if length > max_seq_length:
        for indiv_message in message_log:
            indiv_message["token_ids"] = indiv_message["token_ids"][
                : min(4, max_seq_length // len(message_log))
            ]
        loss_multiplier = 0.0

    output: DatumSpec = {
        "message_log": message_log,
        "length": length,
        "extra_env_info": extra_env_info,
        "loss_multiplier": loss_multiplier,
        "idx": idx,
    }
    if "task_name" in datum_dict:
        output["task_name"] = datum_dict["task_name"]
    return output

4. Configuration 파일 (YAML)

grpo_nanov3.yaml 기반의 전체 설정 파일입니다. 실제 사용 전 수정해야 할 필드는 아래를 참고하세요.

│   ├── dtensor_cfg   → PyTorch FSDP2 기반 백엔드 (소·중형 모델 권장)
│   ├── megatron_cfg  → Megatron 백엔드 (대형 모델, enabled: true 시 dtensor 무시)

자주 수정하는 필드

6.2 KL Penalty 및 Reward 조정

KL penalty는 정책이 참조 모델에서 너무 벗어나지 않도록 제약합니다. 값이 너무 크면 학습이 느려지고, 0이면 순수 GRPO입니다.

bash
Copy

uv run python examples/run_grpo.py --config my_grpo.yaml \
  loss_fn.reference_policy_kl_penalty=0

# KL 적용 (정책 안정화)
uv run python examples/run_grpo.py --config my_grpo.yaml \
  loss_fn.reference_policy_kl_penalty=0.05 \
  loss_fn.reference_policy_kl_type=k3    # k1, k2, k3 중 선택

# ratio clipping 범위 조정 (기본 0.2 ~ 0.28)
uv run python examples/run_grpo.py --config my_grpo.yaml \
  loss_fn.ratio_clip_min=0.1 \
  loss_fn.ratio_clip_max=0.2

6.3 커스텀 보상 함수

NeMo RL의 보상 함수는 EnvironmentInterface를 상속하는 Ray remote 클래스로 구현합니다. step()이 핵심 메서드로, message_log_batch(대화 히스토리)와 metadata(DatumSpec의 extra_env_info)를 받아 EnvironmentReturn을 반환합니다.

Nemotron 3 학습에 실제 사용된 MathMultiRewardEnvironment는 세 가지 보상을 동시에 계산합니다 — 정답 일치, 정수 포맷, XML 구조 준수. 이를 참고한 커스텀 환경 예시입니다.

python
Copy
# my_env/format_reward_env.py
import re
import torch
import ray
from nemo_rl.environments.interfaces import EnvironmentInterface, EnvironmentReturn
from nemo_rl.data.interfaces import LLMMessageLogType

# ① 보상 계산 Worker — Ray remote가 아닌 일반 클래스
class FormatRewardWorker:
    """
    Nemotron 3의 HFMultiRewardVerifyWorker 패턴 참고.
    <think>...</think>\n<answer>...</answer> 포맷 준수 여부를 검사.
    """
    FORMAT_PATTERN = re.compile(
        r"^<think>.*?</think>\n<answer>(.*?)</answer>$",
        re.DOTALL,
    )

    def verify(
        self,
        pred_responses: list[str],
        ground_truths: list[str],
    ) -> tuple[list[float], list[float]]:
        correctness, format_scores = [], []
        for response, gt in zip(pred_responses, ground_truths):
            m = self.FORMAT_PATTERN.match(response.strip())
            fmt = 1.0 if m else 0.0
            answer = m.group(1).strip() if m else ""
            correct = 1.0 if answer == gt.strip() else 0.0
            format_scores.append(fmt)
            correctness.append(correct)
        return correctness, format_scores


# ② 환경 클래스 — @ray.remote 필수
@ray.remote(max_restarts=-1, max_task_retries=-1)
class FormatRewardEnvironment(EnvironmentInterface):

    def __init__(self, cfg: dict):
        self.worker = FormatRewardWorker()

    def step(
        self,
        message_log_batch: list[LLMMessageLogType],
        metadata: list[dict],
    ) -> EnvironmentReturn:
        # 마지막 assistant 응답 추출 (MathEnvironment와 동일한 패턴)
        responses = [
            "".join(
                str(m["content"])
                for m in conv
                if m["role"] == "assistant"
            )
            for conv in message_log_batch
        ]
        ground_truths = [meta["ground_truth"] for meta in metadata]

        correctness, fmt_scores = self.worker.verify(responses, ground_truths)

        # 다중 보상: shape [B, 2]  (correctness, format)
        rewards = torch.tensor(
            list(zip(correctness, fmt_scores)), dtype=torch.float32
        )
        done = torch.ones(len(responses))

        return EnvironmentReturn(
            observations=[{"role": "environment", "content": ""} for _ in responses],
            metadata=metadata,
            next_stop_strings=[None] * len(responses),
            rewards=rewards,
            terminateds=done,
            answers=None,
        )

    def global_post_process_and_metrics(self, batch):
        return batch, {}

다중 보상 (Multi-Reward): rewards를 shape [B, N]으로 반환하면 N개의 보상 컴포넌트를 독립적으로 추적할 수 있습니다. Nemotron 3는 이 방식으로 정답·포맷·정수 여부를 분리해 학습했습니다.
단일 보상이면 torch.tensor(correctness)로 shape [B]를 반환하면 됩니다.

구현한 환경을 학습 스크립트에서 직접 전달하거나 YAML env 섹션에 연결합니다. 자세한 등록 방법은 공식 GRPO 가이드 및 math_environment.py 소스를 참고하세요.

7.2 주요 모니터링 메트릭

아래 메트릭들은 GRPO 학습 안정성과 수렴 여부를 진단하는 데 사용됩니다.

메트릭	설명	권장 범위
token_mult_prob_error	곱셈 토큰 확률 오차 (Multiplicative Token Probability Error)
생성(generation) 시점의 토큰 확률과 학습(policy) 시점의 토큰 확률 간의 곱셈적 상대 오차를 측정합니다. 두 확률이 일치할수록 학습이 안정적으로 진행됩니다.	< 1–2%
gen_kl_error
policy_kl_error
js_divergence_error	KL 발산 오차 (KL Divergence Error)
gen_kl_error: 생성 분포 P_gen과 정책 분포 P_policy 사이의 KL 발산
policy_kl_error: 정책 분포 P_policy와 참조 분포 P_ref 사이의 KL 발산
js_divergence_error: 두 분포 간 대칭적 유사도를 나타내는 Jensen-Shannon 발산
값이 작을수록 생성·학습·참조 분포가 일치하며 학습이 안정적입니다.	< 1e-3
sampling_importance_ratio	샘플링 중요도 비율 (Sampling Importance Ratio)
정책 확률과 생성 확률의 비율로, 생성 시점과 학습 시점 사이의 분포 이동을 보정하는 데 사용됩니다. 비율이 1에 가까울수록 두 시점의 분포가 일치합니다.	≈ 1.0
approx_entropy	엔트로피 (Entropy)
정책 분포의 근사 엔트로피로, 정책 붕괴(policy collapse) 여부를 감지합니다. 엔트로피가 급격히 낮아지면 모델이 특정 출력으로 수렴하며 다양성을 잃는 신호입니다.	안정적으로 유지
