NVIDIA NIM Quick Start Guide
NIM (NVIDIA Inference Microservices) — 사전 최적화된 AI 모델을 REST API로 즉시 사용할 수 있는 서비스.
build.nvidia.com 클라우드 API 또는 온프레미스 Docker 배포 모두 동일한 OpenAI 호환 API로 사용합니다.
1. 사전 준비사항 및 설정
NVIDIA 개발자 계정 생성
developer.nvidia.com 접속
우측 상단 Join 클릭 → 이메일로 가입 또는 Google/GitHub 계정으로 소셜 로그인
이메일 인증 완료 후 build.nvidia.com에서 동일 계정으로 로그인 가능

계정 하나로 통합 관리: NVIDIA Developer 계정은 NIM API(build.nvidia.com), NGC(ngc.nvidia.com), NVIDIA 포럼 등 NVIDIA 서비스 전체에서 공용으로 사용됩니다.

NIM 엔드포인트 선택

NIM은 두 가지 방식으로 사용할 수 있습니다. 온프레미스 배포를 권장합니다.

✅ 권장: 온프레미스 NIM 호스팅

NIM 컨테이너를 직접 실행하면 데이터가 로컬에 유지되고 네트워크 레이턴시가 없어집니다.

사전 요구사항:

NVIDIA GPU (A100/H100 이상을 권장)
Docker + NVIDIA Container Toolkit
NGC 계정 및 NGC API Key
bash
Copy
# NGC 로그인
docker login nvcr.io -u '$oauthtoken' -p <NGC_API_KEY>

# NIM 컨테이너 실행 (Nemotron-3-Nano-30B)
docker run -it --rm --gpus all \
  -e NGC_API_KEY=$NGC_API_KEY \
  -v ~/.cache/nim:/opt/nim/.cache \
  -p 8000:8000 \
  nvcr.io/nim/nvidia/nemotron-3-nano-30b-a3b:latest

2. Nemotron-3

Nemotron-3는 에이전틱 AI 애플리케이션을 위해 설계된 NVIDIA의 오픈 모델 Family입니다. Mamba-Transformer와 Mixture-of-Experts(MoE)를 결합한 하이브리드 아키텍처로 표준 Transformer 대비 높은 처리량을 제공하며, 최대 1M 토큰 컨텍스트와 Inference-Time Budget Control을 지원합니다. (NVIDIA Developer Blog)

3. 기본 Chat Completion

NIM Chat API는 OpenAI와 완전히 동일한 인터페이스입니다. 기존 OpenAI 코드의 base_url과 api_key만 변경하면 바로 사용할 수 있습니다.

Python
python
Copy
import os
import json
import numpy as np
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()

# 온프레미스 NIM (권장)
client = OpenAI(
    base_url="http://localhost:8000/v1",
    api_key="no-key",  # 로컬에서는 인증 불필요
)

# Cloud API 사용 시 위 코드를 아래로 교체
# client = OpenAI(
#     base_url="https://integrate.api.nvidia.com/v1",
#     api_key=os.environ["NVIDIA_API_KEY"],
# )

CHAT_MODEL = "nvidia/nemotron-3-nano-30b-a3b"
# CHAT_MODEL = "nvidia/nemotron-3-super-120b-a12b"  # 고성능

USE_STREAM = True  # False: 일반 응답, True: 스트리밍

# --- 단일 턴 ---
messages = [
    {"role": "system", "content": "당신은 친절한 AI 어시스턴트입니다."},
    {"role": "user",   "content": "NIM이 뭐야? 두 문장으로 설명해줘."},
]

if USE_STREAM:
    stream = client.chat.completions.create(
        model=CHAT_MODEL, messages=messages, temperature=0.5, max_tokens=256, stream=True,
    )
    print("Assistant: ", end="", flush=True)
    for chunk in stream:
        if chunk.choices and chunk.choices[0].delta.content:
            print(chunk.choices[0].delta.content, end="", flush=True)
    print()
else:
    response = client.chat.completions.create(
        model=CHAT_MODEL, messages=messages, temperature=0.5, max_tokens=256,
    )
    print(response.choices[0].message.content)
    print(f"\n[토큰] 입력={response.usage.prompt_tokens}, 출력={response.usage.completion_tokens}")

# --- 멀티턴 (히스토리 직접 관리) ---
history = [{"role": "system", "content": "당신은 친절한 AI 어시스턴트입니다."}]

def chat(user_input: str) -> str:
    history.append({"role": "user", "content": user_input})
    resp = client.chat.completions.create(
        model=CHAT_MODEL, messages=history, temperature=0.5, max_tokens=256
    )
    reply = resp.choices[0].message.content
    history.append({"role": "assistant", "content": reply})
    return reply

print("Turn 1:", chat("내 이름은 준호야."))
print("\nTurn 2:", chat("내 이름이 뭐라고 했지?"))  # 이전 대화를 기억

4. Reasoning 사용법

Nemotron 모델은 Thinking Mode를 지원합니다. 최종 답변 전에 단계별 추론 과정을 거쳐 더 정확한 결과를 냅니다.

extra_body 파라미터	설명
enable_thinking: True	추론 모드 활성화
reasoning_budget	추론에 사용할 최대 토큰 수 (기본 4096, 최대 16384)

스트리밍 응답 구조:

delta.reasoning_content → 추론 과정 (thinking)
delta.content → 최종 답변
python
Copy
print("=== 추론 과정 ===\n")
in_answer = False

stream = client.chat.completions.create(
    model=CHAT_MODEL,
    messages=[{"role": "user", "content": "닭과 토끼가 합쳐서 20마리, 다리 합계가 56개일 때 각각 몇 마리?"}],
    temperature=1.0,
    max_tokens=16384,
    extra_body={
        "reasoning_budget": 8192,
        "chat_template_kwargs": {"enable_thinking": True},
    },
    stream=True,
)

for chunk in stream:
    if not chunk.choices:
        continue
    delta = chunk.choices[0].delta
    reasoning = getattr(delta, "reasoning_content", None)
    content = delta.content

    if reasoning:
        print(reasoning, end="", flush=True)
    elif content:
        if not in_answer:
            print("\n\n=== 최종 답변 ===\n")
            in_answer = True
        print(content, end="", flush=True)

print()
5. Function Calling (Tool Use)

LLM이 외부 함수를 호출해야 하는 시점을 판단하고 인자를 JSON으로 생성합니다.

Function Calling 흐름
사용자 메시지
  → LLM: tool_calls 반환 (어떤 함수를, 어떤 인자로)
      → 클라이언트: 실제 함수 실행 후 결과를 messages에 추가
          → LLM: 결과를 바탕으로 최종 답변 생성
python
Copy
# 1. 도구 정의
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "특정 도시의 현재 날씨를 조회합니다.",
        "parameters": {
            "type": "object",
            "properties": {
                "location": {"type": "string", "description": "도시 이름 (예: 서울, 부산, Tokyo)"}
            },
            "required": ["location"],
        },
    },
}]

# 2. 실제 함수 구현 (Mock)
def get_weather(location: str) -> str:
    mock = {"서울": "맑음, 18°C, 습도 55%", "부산": "흐림, 22°C, 습도 70%"}
    return mock.get(location.lower(), f"{location}: 정보 없음")

# 3. 1차 요청
messages = [{"role": "user", "content": "서울 날씨가 어때?"}]
response = client.chat.completions.create(
    model=CHAT_MODEL, messages=messages, tools=tools, tool_choice="auto", max_tokens=256,
)
msg = response.choices[0].message

# 4. tool_calls 처리 → 5. 2차 요청
if msg.tool_calls:
    for tool_call in msg.tool_calls:
        fn_args = json.loads(tool_call.function.arguments)
        result = get_weather(**fn_args)
        messages.append(msg)
        messages.append({"role": "tool", "tool_call_id": tool_call.id, "content": result})
    final = client.chat.completions.create(model=CHAT_MODEL, messages=messages, max_tokens=256)
    print(f"최종 답변: {final.choices[0].message.content}")

API 파라미터 요약
파라미터	설명	기본값
temperature	창의성 (0=결정적, 2=창의적)	1.0
max_tokens	최대 출력 토큰 수	모델마다 다름
top_p	누클리어스 샘플링	1.0
stream	스트리밍 활성화	False
n	응답 후보 수	1
stop	생성 중단 문자열	null
seed	재현 가능한 출력을 위한 시드 값	null
frequency_penalty	이미 나온 토큰 억제 (-2~2)	0.0
presence_penalty	새로운 토픽 유도 (-2~2)	0.0
response_format	출력 형식 지정 (JSON 강제 등)	null
tools	Function Calling 도구 목록	null
tool_choice	도구 호출 방식	"auto"
