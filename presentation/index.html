Datasets That
Don't Go Stale.

Every benchmark saturates. We built a pipeline that evolves existing datasets to match the current capability of the model — automatically, verifiably, and on its own schedule.

Seed → Synthesizer → Evaluator → Feedback Verifiable reward · no LLM judge Nemotron-3-Nano · GLM-5.1 Heuristic ASR — on GLM-5.1

Why

The effort that goes into a good benchmark is real. The half-life isn't.

Building a good dataset is hard. Difficulty, diversity, label accuracy — meeting all of these conditions takes researchers months, sometimes years of careful curation.

Then models catch up. Benchmarks saturate. Training sets start rehearsing what the model already knows. The standard response is to start from scratch — and throw the craft away.

We don't throw it away.
We evolve it to match the current capability of the model.

The Pipeline

One loop. The seed never changes; the strategy learns.

Tier 1
Heuristic
Fixed rule-based templates. No LLM in the rewrite. Deterministic baseline.
Tier 2
Evol-Instruct
LLM rewrites with a fixed strategy. One call per seed, no feedback.
Tier 3 · ours
Self-Evolving
The strategy text itself is optimized from evaluation feedback.
